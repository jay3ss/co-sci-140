1. Encapsulation
    The combination of code and data in a single object. This is a method of hiding the inner details. Functions encapsulate behavior and objects encapsulate data and behavior [1].

  2. Classes support encapsulation and information hiding by

    1. Combining both data (member attributes) and Functions (member) methods into one object (encapsulation).

    2. Information hiding is an object's ability to hide its data from outside the object. C++ classes do this by using access specifiers. The access specifier private prevents code outside of the object from modifying or even reading any attributes or functions declared private. This prevents corruption of the object's data and outside code knows nothing about the inner details of the object.

  3. Class member variables declared as static are accessible to all instances of that class. No matter how many instances of a class there are, there will only be one copy of static member variables. You must remember to have a declaration of the static member variable that's outside of the class in order for it to be created in memory. An example of this is counting the number of instances of a class. For example, 

    // MyClass.h
class MyClass
{
private:
    static int numObjects;
public:
    MyClass()
    { numObjects++; }

    int getNumObjects() const
    { return numObjects; }
};

// Define in global scope of program
int MyClass::numObjects = 0;

// In main function of program
MyClass myClass1;
MyClass myClass2;

// Both calls to getNumObjects will result in 2
cout << "Number of objects of type MyClass: " 
     << myClass1.getNumObjects();

cout << "Number of objects of type MyClass: " 
     << myClass2.getNumObjects();

    Class member functions declared as static can be called before any instances of the class are instantiated. and can only operate on static member variables. When using a static member function, make sure to use the scope resolution operator (::) instead of the dot operator. Also note that the class name is used instead of an instance when calling the function as well. The above example can be modified to use a static member function instead of a variable. 

    // MyClass.h
class MyClass
{
private:
    static int numObjects;
public:
    MyClass()
    { numObjects++; }

    static int getNumObjects()
    { return numObjects; }
};

// Define in global scope of program
int MyClass::numObjects = 0;

// In main function of program

cout << "Number of objects of type MyClass: "
     << MyClass::getNumObjects();
// The output of the cout statement above is
// Number of objects of type MyClass: 0

// Instantiate two MyClass objects
MyClass myClass1;
MyClass myClass2;

cout << "Number of objects of type MyClass: " 
     << MyClass::getNumObjects();
// The output of the cout statement above is
// Number of objects of type MyClass: 2
