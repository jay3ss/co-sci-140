1. Many times, an will be thrown when an error occurs. If not handled, the exception will cause the program to crash. To handle the exception, a try-catch statement must be used to capture the exception and to gracefully handle the error. The exception that is thrown can be a string or a class. Both are shown in the example below 

    try
{
    // If an exception is thrown in the try block, then any
    // code remaining to be executed after the exception is
    // thrown is skipped and the catch block is executed
    quotient = divide(num1, num2);
    cout << "The quotient is " << quotient << endl;
}
catch (string exceptionString)
{
    // If the exception is a string, then this catch block
    // will be executed
    // If there is no exception, then this block is not executed
    cout << exceptionString;
}

// Object-oriented method
try
{
    // A division object is instantiated
    Division divide(num1, num2);
    cout << "The quotient is " << divide.quotient << endl;
}
catch(Division::DivisionByZeroException)
{
    // A DivisionByZeroException was thrown
    cout << "Error: Division by zero was attempted.\n";
}
// The program can continue because the exception was caught

    Multiple exceptions can be caught by using multiple catch blocks. Also, exceptions can be rethrown using the throw statement inside of a catch block so that an outer throw block can catch the exception and recover.

  2. A function template is a generic function that allows a function to work with (almost) any data type. An example is the absolute value function. One way to define an absolute value function that works with multiple data types is to create multiple overloaded functions. This works for a couple of data types, but can quickly get out of hand. Therefore, by using a function template, the function only has to be written (and tested!) once. For example, 

    // Using overloaded functions
int absval(int num)
{
    if (num < 0)
    {
        return -num;
    }
    else
    {
        return num;
    }
}

double absval(double num)
{
    if (num < 0)
    {
        return -num;
    }
    else
    {
        return num;
    }
}

// Using a function template
template<class T>
T absval(T num)
{
    if (num < 0)
    {
        return -num;
    }
    else
    {
        return num;
    }
}

    Similar to function templates, class templates allow generic classes to be created that work with multiple data types instead of a bespoke class for each data type. An example is with a list. A list stores data and this data is of the same data type. Therefore, to store multiple data types multiple list classes must be created. If, however, a class template is used then only one generic list class will need to be created.
