1. Many times, an will be thrown when an error occurs. If not handled, the exception will cause the program to crash. To handle the exception, a try-catch statement must be used to capture the exception and to gracefully handle the error. The exception that is thrown can be a string or a class. Both are shown in the example below 

    try
{
    // If an exception is thrown in the try block, then any
    // code remaining to be executed after the exception is
    // thrown is skipped and the catch block is executed
    quotient = divide(num1, num2);
    cout << "The quotient is " << quotient << endl;
}
catch (string exceptionString)
{
    // If the exception is a string, then this catch block
    // will be executed
    // If there is no exception, then this block is not executed
    cout << exceptionString;
}

// Object-oriented method
try
{
    // A division object is instantiated
    Division divide(num1, num2);
    cout << "The quotient is " << divide.quotient << endl;
}
catch(Division::DivisionByZeroException)
{
    // A DivisionByZeroException was thrown
    cout << "Error: Division by zero was attempted.\n";
}
// The program can continue because the exception was caught
